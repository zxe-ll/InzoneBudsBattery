# 実機検証結果

検証日: 2026-07-15

対象: Sony INZONE Buds（WF-G700N）USBトランシッター

## HIDインターフェース

VID `054C` / PID `0EC2`で5個のHIDインターフェースを検出しました。

| Collection | Input Report長 | 用途 |
| --- | ---: | --- |
| `col01` | 62 | 未使用 |
| `col02` | 2 | 未使用 |
| `col03` | 64 | バッテリーレポート |
| `col04` | 0 | Input Reportなし |
| `col05` | 8 | 未使用 |

`col03`は共有アクセスで開くことができ、INZONE Hub起動中も同時に読み取れました。

## バッテリーレポート

- 64バイト
- ヘッダー: `02 12 04`
- 左残量: byte 14
- 右残量: byte 16
- ケース残量: byte 18
- チェックサム: `byte[19] = sum(byte[5] ... byte[18]) mod 256`

INZONE Hub画面のL/R/CaseとProbeの値が一致することを確認しました。チェックサムも取得した実機レポートで一致しています。

## INZONE Hubとの共存

- Hub起動中もReport長0の`col04`を除くインターフェースを共有オープンできた。
- HubとProbeでL 100% / R 100% / Case 93%が一致した。
- Hub起動中のトランシッター切断・再接続後、HubとProbeの両方が復帰した。
- Hub終了・再起動後も両方が正常動作した。

## Dalamudプラグイン

バージョン0.2.1.0を約20分間連続動作させ、例外や切断を確認しませんでした。

自発的なInterrupt Input Reportは約2～4分間隔で届き、次の残量変化を取得しました。

| 経過 | Left | Right | Case | Checksum |
| --- | ---: | ---: | ---: | --- |
| 初回 | 92% | 93% | 93% | 一致 |
| 約4分30秒後 | 91% | 93% | 93% | 一致 |
| 約2分5秒後 | 91% | 92% | 93% | 一致 |

標準の`HidD_GetInputReport`は3分ごとに呼び出し自体は成功しましたが、Report ID `02`以外がゼロの非バッテリーレポートを返しました。Interrupt Input Reportの継続待受には影響せず、その後もバッテリーレポートを受信できました。

## 既知の制約

- 起動直後は最初の自発レポートが届くまで残量を表示できない。
- ケース残量は、ケースとイヤホンが最後に通信した時点の値である可能性がある。
- 標準の`HidD_GetInputReport`ではバッテリー値を強制取得できない。

## v0.2.2.0 片耳対応

片耳使用時に自発レポートが停止する事象への対策として、Sony HCIの電池イベントID `0x04`を要求する64バイトOutput Reportを追加しました。接続直後と設定間隔ごとに要求し、3秒以内に届く既存形式の`02 12 04`応答を解析します。

2026-07-19のネイティブ検証では、`col03`がInput/Outputとも64バイト、Usage Page `FF04`であることを確認しました。INZONE Hub起動中に共有Read/Writeオープンし、電池GETを送信すると即座に応答しました。片耳状態で次の値を取得しています。

```text
Left:  0xFF（未接続）
Right: 82%
Case:  90%
```

`0xFF`は異常値ではなく未接続側を表すため、v0.2.2.0では`null`へ変換し、接続中の片耳とケースを更新します。右耳だけを装着した実機確認により、byte 14が左、byte 16が右であることも確定しました。

実機では次を確認する必要があります。

- Dalamudプラグイン上で設定間隔ごとに片耳残量が更新される。
- INZONE Hub起動中に長時間の定期要求が継続して成功する。
- Hubとの同時利用で設定変更、切断、クラッシュなどの副作用がない。
- 書込み不可または応答なしの場合に、自発レポート待受が継続する。
